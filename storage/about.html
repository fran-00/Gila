About Gila

What is Gila?

Gila is a client that provides a graphical interface to connect to generative AI language models.

How to use Gila?

API Keys

In order to use the program, you need to obtain at least an API key: this is a unique code that is assigned to an application or web service to allow access and interaction with the provider. Once you have entered an API key into the program, you can start a new conversation with one of the available language models. Using some services involves costs.

Tokens

Tokens are minimal units of character sequences in a text. Large language models analyze a text by dividing it into tokens and from the statistical relationships that exist between the character sequences they are able to produce a coherent text in response. In English a token is equivalent to about 4 characters.

The program will show the number of tokens used for a given conversation: how many tokens constitute the question, how many tokens the prompt and response consist of, and the total number of tokens. Before sending a message, the program will show you an estimate of the number of tokens you are adding, but this may not be accurate because the division of words into tokens varies depending on the model.

The Sidebar

From the sidebar you can see two tabs: Saved Chats and Settings.

Saved Chats

Conversations are automatically saved in the sidebar: the name assigned is a simple 10-letter ID that serves to uniquely identify the conversation, but it is possible to rename them to distinguish them more easily. You can also delete them.

Settings

You can change conversation settings such as model, temperature, maximum tokens and system message. The settings will automatically be applied every time you start a new chat.

  • Model - Here you can choose the language model to chat with from a drop-down menu. If you select a model that depends on a service for which you have not entered an API key, the program will ask you to enter one when you start a new conversation.
  • Temperature - Represents the model's degree of creativity: a higher temperature means more creative and unpredictable responses. Setting high values for this parameter can give strange results...
  • Max Tokens - The maximum number of tokens to generate before stopping. Any model can stop before reaching this maximum number and decides autonomously how many tokens to use. Different models have different maximum values for this parameter: the upper value of the bar will automatically change every time you select a new model.
  • System Message - An optional instruction that guides the AI's behavior and context to make the output more effective. With this message you can define AI's personality, set rules and restrictions, provide task guidelines... For example, you might ask to roleplay a character from a book or to only talk in rhymes. You could achieve the same results with a simple direct request in the chat, but a system message is usually more effective.

The toolbar

From the toolbar you can perform these actions:

  1. Export the current conversation as a .txt or .docx file.
  2. Add API keys to communicate with services.
  3. Open this window with program information.

Issues

It may happen that the program crashes, especially due to connection problems with the web services needed for it to work: in this case, just close Gila and restart it.
